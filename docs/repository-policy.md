# Repository Policy

This repository is intentionally limited.

It is a public showcase for the product, not the production source repository.

## Allowed

- Product introduction
- Public screenshots
- Public roadmap
- Public changelog
- High-level methodology
- Contact information

## Not Allowed

- Production source code
- Backend code
- Account system code
- Payment integration code
- Crawler logic
- API keys
- SMTP credentials
- Server passwords
- Full company database
- Paid content
- Deployment scripts
- Private operational notes

## Why

The project contains proprietary product logic, collected data, and operational configuration. Those assets should remain private while the public repository acts as a credible product record and promotion page.
